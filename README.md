## Explica qué es Github y qué relación tiene con Git.
GitHub es una plataforma en línea que permite alojar y gestionar proyectos de software utilizando Git. Su función principal es facilitar el almacenamiento del código en la nube y el trabajo colaborativo entre desarrolladores. A través de GitHub, los usuarios pueden compartir sus proyectos, revisar cambios y trabajar de manera conjunta desde distintos lugares.

Git es un sistema de control de versiones que se encarga de registrar los cambios realizados en los archivos de un proyecto a lo largo del tiempo. Permite volver a versiones anteriores, trabajar de forma paralela con otras personas y mantener un historial detallado de las modificaciones. La relación entre Git y GitHub es que Git es la herramienta que gestiona las versiones del código, mientras que GitHub es la plataforma que utiliza Git para almacenar los repositorios de forma remota y facilitar el trabajo en equipo.

## Describe los tipos de repositorios que existen en Github.

En GitHub existen los siguientes tipos de repositorios:

- **Repositorios públicos**:
	Son accesibles para cualquier persona. Cualquier usuario puede ver y descargar el dontenido del repositorio, aunque solo quienes tengan permisos pueden modificarlo. Este tipo de repositorios es común en proyectos de código abierto, ya que fomenta la colaboración, el aprendizaje y la participación de la comunidad.

- **Repositorios privados**:
	Solo pueden ser vistos y utilizados por las personas autorizadas por el propietario del repositorio. Ofrecen mayor seguridad y control sobre el acceso al código, por lo que son ideales para proyectos personales, académicos o empresariales que requieran confidencialidad.
